shared `personaVersion` audit tag. The clinician learns the same trust
signal across multiple surfaces — the brief on `/prepare`, the handout in
the visit viewer, the Ask sheet — so when Sprint 0.13's case-routing panel
shows up at review time, she is already a known colleague.

## Design

### Pre-visit brief — `/prepare/[noteId]`

The brief card's header today is anonymous (e.g. *"Patient brief"* or just
the patient's name). Change to:

```
✨ Miss Cleo's read on James Park
Last seen 6 weeks ago — Hypertension follow-up
...
```

- Use the `Sparkles` icon prefix (already used by `CopilotShell`).
- Import `COPILOT_DISPLAY_NAME` from `@/services/copilot/persona` — never
  hardcode the literal string.
- The body of the brief (the prior-context content) is unchanged.

### Visit viewer's Handout tab

The Handout tab inside `/visits/[noteId]` renders the `NoteArtifact` rows
generated by the post-sign-artifacts worker. Section titles change:

- *"Patient handout"* → *"✨ Miss Cleo's draft patient handout"*
- *"Referral letter"* → *"✨ Miss Cleo's draft referral letter"*

The LLM-generated body content is unchanged. Only the section titles.

### Audit metadata on workers

Three workers emit AI-authored audit rows. Each gains `personaVersion`
metadata.

| Worker | Audit action | Metadata addition |
|--------|--------------|-------------------|
| `note-brief/handler.ts` | `BRIEF_GENERATED` | `personaVersion: PERSONA_VERSION` |
| `post-sign-artifacts/handler.ts` | `POST_SIGN_ARTIFACT_GENERATED` | `personaVersion: PERSONA_VERSION` |
| `ai-generation/handler.ts` | `NOTE_GENERATION_STARTED`, `SECTION_GENERATED`, `NOTE_GENERATION_COMPLETED` | `personaVersion: PERSONA_VERSION` (each one) |

Import `PERSONA_VERSION` from `@/services/copilot/persona`.

This unifies the regulator / auditor query: filter `AuditLog` rows by
`metadata.personaVersion = 'miss-cleo-v1'` + `patientId = X` → a full
reconstruction of Miss Cleo's actions for that patient. The same query
already works for `COPILOT_ASK_ANSWERED` and `COPILOT_BEACON_OPENED` (Unit
42 spec). This sprint completes the coverage.

## Three-lens

- **Clinician** — same content, same workflow. The attribution makes the
  trust relationship explicit: *"Miss Cleo read this chart for me."*
- **Compliance** — `personaVersion` filter answers *"what did the AI do for
  this patient?"* in one query, across all surfaces.
- **Auditor** — every AI-authored artifact is now traceable to the persona
  version that generated it; useful for future model rollouts and
  calibration analysis.
